## API Geliştirme Rehberi

Modern e-ticaret sistemleri için API tasarım prensipleri ve best practice'ler.

---

## RESTful API Tasarımı

### Endpoint Yapısı

```
/api/v1/products          # Ürünler
/api/v1/products/{id}     # Tek ürün
/api/v1/categories        # Kategoriler
/api/v1/orders            # Siparişler
/api/v1/customers         # Müşteriler
```

### HTTP Metodları

| Metod | Kullanım | Örnek |
|-------|----------|-------|
| **GET** | Veri okuma | Ürün listesi |
| **POST** | Veri oluşturma | Sipariş oluştur |
| **PUT** | Tam güncelleme | Ürün güncelle |
| **PATCH** | Kısmi güncelleme | Stok güncelle |
| **DELETE** | Silme | Ürün sil |

## E-Ticaret API Modülleri

### Ürün API

```
GET    /products              # Liste
GET    /products/{id}         # Detay
POST   /products              # Oluştur
PUT    /products/{id}         # Güncelle
DELETE /products/{id}         # Sil
GET    /products/{id}/variants # Varyantlar
```

### Sipariş API

```
GET    /orders                # Liste
GET    /orders/{id}           # Detay
POST   /orders                # Oluştur
PATCH  /orders/{id}/status    # Durum güncelle
POST   /orders/{id}/cancel    # İptal
```

### Sepet API

```
GET    /cart                  # Sepeti getir
POST   /cart/items            # Ürün ekle
PUT    /cart/items/{id}       # Miktar güncelle
DELETE /cart/items/{id}       # Ürün çıkar
DELETE /cart                  # Sepeti temizle
```

## Kimlik Doğrulama

### JWT Token

```
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
```

**Token Yapısı:**
- Access Token (kısa ömürlü)
- Refresh Token (uzun ömürlü)

### API Key

```
X-API-Key: your-api-key-here
```

**Kullanım:** B2B entegrasyonlar, webhook'lar

## Sayfalama ve Filtreleme

### Sayfalama

```
GET /products?page=1&limit=20

Response:
{
  "data": [...],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 150,
    "totalPages": 8
  }
}
```

### Filtreleme

```
GET /products?category=123&minPrice=100&maxPrice=500&inStock=true
```

### Sıralama

```
GET /products?sort=price&order=asc
GET /products?sort=-createdAt  # - prefix for desc
```

## Hata Yönetimi

### Standart Hata Formatı

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Geçersiz istek",
    "details": [
      {
        "field": "email",
        "message": "Geçerli bir email adresi giriniz"
      }
    ]
  }
}
```

### HTTP Durum Kodları

| Kod | Açıklama |
|-----|----------|
| 200 | Başarılı |
| 201 | Oluşturuldu |
| 400 | Geçersiz istek |
| 401 | Yetkisiz |
| 403 | Yasaklı |
| 404 | Bulunamadı |
| 422 | Doğrulama hatası |
| 500 | Sunucu hatası |

## Webhook Entegrasyonu

### Desteklenen Eventler

| Event | Açıklama |
|-------|----------|
| `order.created` | Yeni sipariş |
| `order.paid` | Ödeme alındı |
| `order.shipped` | Kargoya verildi |
| `product.updated` | Ürün güncellendi |
| `stock.low` | Düşük stok uyarısı |

### Webhook Payload

```json
{
  "event": "order.created",
  "timestamp": "2026-01-24T12:00:00Z",
  "data": {
    "orderId": "ORD-12345",
    "total": 1500.00
  }
}
```
