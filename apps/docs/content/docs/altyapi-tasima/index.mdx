# Altyapı Taşıma

Mevcut e-ticaret platformunuzdan memnun değil misiniz? Yanlış karar veren binlerce işletme gibi siz de **altyapı taşıma** sürecindesiniz. Bu bölüm, geçişi en az kayıpla yapmanız için hazırlanmıştır.

## Taşıma Süreci

### 1. Mevcut Durum Analizi

Taşıma öncesi mutlaka belgeleyin:

- [ ] Tüm URL yapısı (sitemap)
- [ ] Ürün sayısı ve kategoriler
- [ ] Müşteri veritabanı
- [ ] Sipariş geçmişi
- [ ] SEO değerleri (backlink, domain authority)
- [ ] Entegre sistemler

### 2. Yeni Platform Seçimi

Doğru platformu seçin:

- [ ] İhtiyaç analizi
- [ ] Teknik gereksinimler
- [ ] Bütçe belirleme
- [ ] Referans kontrolü
- [ ] Sözleşme incelemesi

### 3. Veri Migrasyonu

Verilerinizi güvenli şekilde taşıyın:

- [ ] Ürün verileri export
- [ ] Müşteri verileri export
- [ ] Sipariş geçmişi (opsiyonel)
- [ ] İçerik migrasyonu
- [ ] Veri doğrulama

## SEO Kaybetmeden Taşıma

<Callout title="Kritik">

SEO değerlerinizi korumak, taşımanın en önemli kısmıdır.

</Callout>

### 301 Yönlendirme

Her eski URL için yeni URL'e yönlendirme:

```
Eski: /eski-urun/urun-adi.html
Yeni: /urun/urun-adi
```

**Önemli kurallar:**
- Her sayfa için tek yönlendirme
- Yönlendirme zinciri oluşturmayın
- 404 hatalarını takip edin

### Search Console İşlemleri

1. Yeni sitemap.xml gönder
2. Adres değişikliği bildir
3. Hata raporlarını takip et
4. Index durumunu izle

## Sözleşme Dikkat Noktaları

### Cayma Şartları

- Bildirim süresi (genelde 30-60 gün)
- Cayma bedeli var mı?
- Veri export ücreti
- Domain transfer şartları

### Veri Sahipliği

Sözleşmede mutlaka olsun:

- ✅ Tüm verilerinizin size ait olduğu
- ✅ İstediğiniz zaman export edebileceğiniz
- ✅ Export formatları (CSV, XML, JSON)
- ✅ Export ücreti (varsa)
